1. 项目概述

招商派单系统是一套面向招商拜访业务的智能任务分配系统。系统根据客户预约信息（类型、时间、园区、Plus次数等）和员工能力（职责、园区、单量容量等），自动将客户拜访任务分配给最合适的员工，同时兼顾通勤距离优化和特殊业务规则（指定人、放弃人、牵手单等）。

核心能力

  • 支持三种客户类型：首访（前道） 项目 回访（后道）

  • Excel 数据导入导出，无缝对接现有工作流

2. 业务背景与派单逻辑

2.1 业务场景

招商团队每天需要拜访大量企业客户，客户按拜访阶段分为三类：

类型说明对应员工职责数据来源
首访（前道）首次拜访新客户前道首访数据.xlsx
项目项目相关拜访项目项目数据.xlsx
回访（后道）后续回访跟进后道回访数据.xlsx

2.2 预约时段

时段时间范围对应员工单量
上午12:00 之前上午单
下午112:00 - 14:30下午单-1
下午214:30 之后下午单-2

2.3 Plus 次数

Plus 次数表示客户被拜访的累计次数，影响派单优先级：

  • Plus0（0次）：新客户，优先级最高
  • Plus1（1次）：回访一次，优先级次之
  • PlusN（2次及以上）：多次回访，优先级最低

派单时先分配 Plus0/Plus1 客户给只具备 Plus0/Plus1 能力的员工，再分配 PlusN 客户。

2.4 派单分组策略

前道 + 项目 → 合并派单
前道单和项目单合并为一个派单池，分配给同时负责"前道"和"项目"职责的员工。支持三种模式：
  • 交叉类型：员工上午接前道单、下午接项目单（或反之），不能两单同类型
  • 同类型：员工上下午接相同类型单
  • 随机：不做类型强制，随机分配（当前默认模式）
后道 → 独立派单
回访（后道）单单独派单，员工上下午都接后道单，不混合前道/项目单。

2.5 距离与通勤要求

场景通勤限制
上午+下午各一单总通勤时间 ≤ 4 小时
全天仅一单上午 ≤ 3h，下午 ≤ 3.5h（建议值）
上午1单+下午2单总通勤 ≤ 4h，下午第二单 ≤ 1h
牵手单不受通勤限制，整组分给同一员工

4.1 完整数据流

阶段一：数据输入
  首访数据.xlsx ──┐
  项目数据.xlsx ──┼──▶ ExcelImporter.importAllData()
  回访数据.xlsx ──┤    ├── 解析字段、时段、Plus次数

6. 派单规则引擎

派单引擎按优先级依次执行以下规则，所有规则必须同时满足才能分配（硬性约束），之后按评分和通勤距离选择最优员工。

规则 1：城市匹配
客户所在城市必须和员工所在城市一致。例：上海客户 → 上海员工。
规则 2：职责匹配
客户类型必须对应员工职责：首访→前道、项目→项目、回访→后道。
规则 3：时段/单量匹配
客户预约时段必须在员工单量容量内。例：上午客户 → 有"上午单"容量的员工。
规则 4：指定人
若客户有指定人，且指定人在候选员工列表中，必须分配给该指定人；若不在列表中，可分配给任何人。
规则 5：放弃人
若客户有放弃人，且放弃人在候选列表中，不能分配给该放弃人。
规则 6：招商园区匹配
客户所属招商园区必须和员工负责的园区一致。
规则 7：Plus 次数匹配
客户的 Plus 等级必须在员工对应职责的 Plus 能力范围内。

6.1 优先级排序

  1. 硬性规则：上述 7 条规则全部通过
  2. 指定人加分：+50 分
  3. Plus 优先级：Plus0 先于 Plus1 先于 PlusN 分配
  4. Plus 能力匹配：优先分配给只具备 Plus0/Plus1 的员工（+20/+15 分）
  5. 负载均衡：已分配单数多的员工扣分（-5/单）
  6. 通勤优化：在满足以上条件后，选择通勤时间最短的组合

6.2 特殊规则

特殊场景处理方式
牵手单备注含"牵手单"的客户整组分配给同一员工，不受通勤限制
指定人（刘帅）江苏镇江项目单必须分给刘帅
放弃人（黄健）江苏徐州项目单不能分给黄健
下午跑2单（姚焕）姚焕有下午单-2容量，可接下午第二单

8. 核心模块说明

8.1 DispatchEngine（派单引擎）

import { DispatchEngine } from './services/dispatch-engine';

const engine = new DispatchEngine({
  frontProjectMode: FrontProjectMode.RANDOM,  // 前道+项目派单模式
  maxTotalCommuteMinutes: 240,                 // 总通勤上限 4h
  enableDistanceOptimization: true,            // 启用距离优化
  allowCommuteOverridePlus: true,              // 允许通勤覆盖Plus优先级
});

const batch = engine.dispatch(customers, employees);
// batch.results     → 派单结果数组
// batch.statistics  → 统计数据
