1. 系统概览

派单系统是一个基于 TypeScript + Express 的 Web 应用，主要提供两个页面：

页面路径功能
Excel 上传派单/ 或 /index.html上传 Excel 文件，执行批量自动派单
选择公司派单看板/match.html左侧勾选公司，AI/规则自动匹配员工，支持增量匹配、一人多单、员工行程

架构示意

用户浏览器 → Nginx :80/443 → Node.js Express :3004 → Excel 数据 / 内存会话
说明：当前 Web 派单使用内存会话（约 30 分钟过期），重启服务后需重新加载数据。DeepSeek API 为可选功能，未配置或余额不足时自动使用本地通勤估算。
在线阅读：服务启动后也可通过浏览器访问 http://你的域名/docs/deploy-guide.html （本地开发：http://localhost:3004/docs/deploy-guide.html）

3. 项目结构与文件清单

部署时，请将整个项目目录上传到服务器，建议路径：/opt/dispatch-system 或 /home/你的用户名/派单系统。

必须上传的文件/目录

路径说明是否必须
src/TypeScript 源代码是（用于构建）
public/前端页面（index.html、match.html 等）是
package.json、package-lock.json依赖声明是
tsconfig.jsonTypeScript 编译配置是
园区数据.xlsx园区主数据是（示例/默认数据）
首访数据.xlsx首访客户至少一份客户表
项目数据.xlsx项目客户至少一份客户表
回访数据.xlsx回访客户至少一份客户表
派单员工表 (1).xls员工基础信息是
.env环境变量（端口、API Key）是（在服务器上创建，勿泄露）

不需要上传（服务器上重新生成）

  • node_modules/ — 在服务器执行 npm install 生成
  • dist/ — 在服务器执行 npm run build 生成
注意：.env 含有 DeepSeek API Key，已在 .gitignore 中忽略。请用 SFTP/SSH 手动上传到服务器，不要提交到 Git 仓库。可参考项目根目录的 .env.example 创建。

4. 上传代码到服务器

方式 A：SCP / SFTP（最简单）

在本地 Windows PowerShell 或终端执行（替换 IP 和路径）：

# 将整个项目文件夹上传到服务器
scp -r "C:\Users\你的用户名\Desktop\派单系统" root@你的服务器IP:/opt/dispatch-system

也可使用 WinSCP、FileZilla 等图形工具，把文件夹拖到服务器 /opt/dispatch-system。

方式 B：Git（适合持续更新）

# 服务器上
cd /opt
git clone 你的仓库地址 dispatch-system
cd dispatch-system

# 单独上传 Excel 和 .env（Git 中通常不包含）
# 用 scp 上传数据文件和 .env

方式 C：压缩包

# 本地打包（排除 node_modules）
# 上传 dispatch-system.zip 到服务器后：
cd /opt
unzip dispatch-system.zip -d dispatch-system
cd dispatch-system

5. 安装依赖与构建

SSH 登录服务器后，进入项目目录执行：

cd /opt/dispatch-system

# 1. 安装生产依赖
npm install --production=false

# 2. 编译 TypeScript
npm run build

# 3. 确认 dist 目录已生成
ls dist/server.js
构建成功后，生产环境使用 npm run start:web 启动（运行 node dist/server.js），无需安装 tsx 到生产常驻进程。

6. 环境变量配置

在项目根目录创建 .env 文件：

cd /opt/dispatch-system
cp .env.example .env
nano .env
变量名说明示例
PORT服务监听端口3004
DEEPSEEK_API_KEYDeepSeek API 密钥（可选）sk-xxxx
DATABASE_URL数据库连接（当前 Web 功能可不依赖）file:./dev.db
不配置 DEEPSEEK_API_KEY 时，系统仍可正常运行，通勤时间使用本地算法估算。配置后可在匹配看板看到「DeepSeek / AI+本地」来源标识。

7. 启动服务

临时启动（测试用）

cd /opt/dispatch-system
npm run start:web

看到以下输出表示成功：

派单系统 Web 服务已启动: http://localhost:3004
  Excel上传派单: http://localhost:3004/
  选择公司派单: http://localhost:3004/match.html

后台启动（简单方式）

nohup npm run start:web > logs/app.log 2>&1 &

建议创建日志目录：mkdir -p logs

8. PM2 进程守护（推荐）

PM2 可在崩溃后自动重启、开机自启，适合生产环境。

# 全局安装 PM2
sudo npm install -g pm2

cd /opt/dispatch-system

# 启动应用
pm2 start dist/server.js --name dispatch-system

# 查看状态
pm2 status
pm2 logs dispatch-system

# 开机自启
pm2 save
pm2 startup
# 按提示执行输出的 sudo 命令

使用 ecosystem 配置文件（可选）

项目可创建 ecosystem.config.cjs：

module.exports = {
  apps: [{
    name: 'dispatch-system',
    script: 'dist/server.js',
    cwd: '/opt/dispatch-system',
    instances: 1,
    autorestart: true,
    watch: false,
    max_memory_restart: '512M',
    env: {
      NODE_ENV: 'production',
      PORT: 3004
    }
  }]
};
pm2 start ecosystem.config.cjs
pm2 save
多实例注意：当前会话数据存在内存中，不要使用 PM2 的 cluster 多实例模式（instances: 1）。否则不同请求可能落到不同进程，导致「数据会话已过期」。

9. Nginx 反向代理与域名

推荐只对外暴露 80/443，Node 服务监听本机 3004。

sudo nano /etc/nginx/sites-available/dispatch
server {
    listen 80;
    server_name dispatch.你的域名.com;   # 或服务器公网 IP

    client_max_body_size 20M;   # 允许上传 Excel

    location / {
        proxy_pass http://127.0.0.1:3004;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

        # 匹配接口可能较慢（通勤计算）
        proxy_read_timeout 120s;
        proxy_connect_timeout 60s;
    }
}
# 启用站点
sudo ln -s /etc/nginx/sites-available/dispatch /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

配置完成后访问：

  • 派单看板：http://dispatch.你的域名.com/match.html
  • Excel 派单：http://dispatch.你的域名.com/

10. HTTPS 证书

有域名时，推荐使用 Let's Encrypt 免费证书：

sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d dispatch.你的域名.com

按提示输入邮箱并同意条款，Certbot 会自动修改 Nginx 配置并配置自动续期。

11. 防火墙与安全

UFW 示例（Ubuntu）

sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
sudo ufw status

不要对公网开放 3004，仅通过 Nginx 反代访问。

安全建议

  • .env 权限设为仅所有者可读：chmod 600 .env
  • 定期更新系统：sudo apt update && sudo apt upgrade
  • 如需限制访问，可在 Nginx 增加 IP 白名单或 HTTP 基本认证
  • 生产环境勿将 DeepSeek Key 写入代码或提交 Git

12. 部署验证

命令行检查

# 服务是否在监听
curl -s http://127.0.0.1:3004/api/sample-data | head -c 200

# 通过 Nginx 检查
curl -s http://dispatch.你的域名.com/match.html | head -c 100

浏览器检查

  1. 打开 /match.html，页面应加载出公司列表（约 55 家，含「演示」标签项）
  2. 勾选几家公司，右侧应出现匹配结果
  3. 点击「一键演示匹配」，应匹配 10 家演示公司
  4. 切换到「员工行程」标签，查看多单行程
  5. 打开首页 /，测试 Excel 上传派单

API 端点一览

方法路径用途
GET/api/sample-data加载整合数据（原始 + 演示）
POST/api/dispatch/select选择公司自动匹配
POST/api/dispatch/validate-pair手动改派校验
POST/api/dispatchExcel 上传批量派单
POST/api/export导出派单结果 Excel

13. 更新与回滚

常规更新流程

cd /opt/dispatch-system

# 1. 拉取/上传新代码
git pull   # 或重新 scp 覆盖

# 2. 安装新依赖（如有）
npm install

# 3. 重新构建
npm run build

# 4. 重启服务
pm2 restart dispatch-system

# 5. 查看日志确认
pm2 logs dispatch-system --lines 50

仅更新 Excel 数据

替换项目根目录下的 xlsx/xls 文件后，重启服务即可。用户刷新页面会重新从 /api/sample-data 加载。

回滚

git checkout 上一个版本号
npm run build
pm2 restart dispatch-system

14. 常见问题

现象可能原因解决办法
EADDRINUSE 端口被占用 3004 已有进程 sudo lsof -i :3004 查 PID 后 kill，或改 .env 中 PORT
页面空白 / 加载失败 服务未启动或 Nginx 配置错误 检查 pm2 status、nginx -t、防火墙
「数据会话已过期」 会话 30 分钟过期或服务重启 刷新页面重新加载数据
公司列表为空 Excel 文件未上传或路径不对 确认根目录有 首访/项目/回访/员工 表，文件名完全一致
通勤显示「本地」而非 AI 未配置 Key 或余额不足 检查 .env 中 DEEPSEEK_API_KEY，重启服务
上传 Excel 报错 413 Nginx 请求体限制 在 Nginx 配置中加 client_max_body_size 20M;
npm run build 失败 Node 版本过低或缺依赖 升级 Node 到 18+，执行 npm install

15. 部署检查清单

上线前逐项确认：

  • 服务器已安装 Node.js 18+ 和 npm
  • 项目代码已上传到 /opt/dispatch-system（或目标目录）
  • Excel 数据文件（园区、首访、项目、回访、员工表）已在项目根目录
  • .env 已创建，PORT 与防火墙策略一致
  • 已执行 npm install 和 npm run build
  • PM2 已启动且 pm2 status 显示 online
  • Nginx 反代配置完成，nginx -t 通过
  • 域名 DNS 已解析到服务器 IP（如使用域名）
  • HTTPS 证书已配置（推荐）
  • 浏览器可打开 /match.html 并正常匹配
  • 已设置 PM2 开机自启 pm2 save && pm2 startup

快速命令备忘

pm2 restart dispatch-system
pm2 logs dispatch-system
pm2 monit
